# Live Trigger 改善メモ

## 2026-03-20

### 1. `app.py` 側で当日データがあっても再取得しないと先に進めない

現象:

- 当日対象の候補抽出データがすでに存在していても、`app.py` 側では再度 `候補抽出` を実行しないと次の確認フローへ進みにくい
- 運用上は「すでにある当日データを読み直して続きから確認したい」場面がある

確認された状況:

- システム再起動後
- 当日分の候補抽出データがファイルとして残っている
- それでも UI 上では再取得が必要なように見え、運用上分かりにくい

改善したいこと:

- 当日分の `watchlist` または関連データが存在する場合、再抽出なしでそのまま読めるようにする
- `既存データを使う / 再抽出する` を UI 上で明示的に選べるようにする
- 再起動後でも「今日のデータがあるので続きから再開可能」と分かる表示を出す

候補案:

- `app.py` 起動時に当日分の `watchlist` を自動検出する
- `既存データを読み込む` ボタンを追加する
- 当日分データの最終更新時刻、対象 profile、件数を表示する
- 当日データが存在する場合は、再抽出を必須にしない

期待する状態:

- 再起動後でも、当日分の候補抽出データが残っていればそのまま確認フローへ進める
- 再抽出が必要な場合だけ明示的に再取得する

### 2. Teleboat 常駐ブラウザのログイン後タブ整理

現象:

- `Teleboat セッション準備` 後、ログイン画面タブが残ったままトップ画面タブも開くことがある
- その状態だと、自動側が古いログイン画面タブを掴む余地があり、確認や実行が不安定になる

現在の運用仕様:

- Teleboat 用の常駐ブラウザは `1ウィンドウ / 1タブ` を基本にする
- ログイン後は、ログイン画面タブを閉じてトップ画面だけ残す
- `Teleboat ログイン確認` は、その状態で実行する

将来の改善余地:

- ログイン成功後に、不要な Teleboat タブを完全自動で整理する
- 「トップ画面タブだけを正」とする選択ロジックをさらに強化する
- UI 上に「常駐ブラウザのタブ数」や「現在つかんでいる URL」を見える化する

期待する状態:

- 人手でタブ整理しなくても、常駐ブラウザは常にトップ画面 1 タブへ収束する
- 自動側がログイン画面タブを誤って使うことがなくなる
