# Project Discovery

This folder is the umbrella concept shelf for logic-discovery assets that do
not belong cleanly to a single runtime project yet.

Use it for:

- zero-base concept families
- cross-project structural hypotheses
- candidate filters that may later be attached to more than one logic
- early-stage ideas that are too important to leave only in one-off reports

Primary evidence currently tends to live in:

- `workspace_codex/analysis/discovery_*`
- `reports/strategies/recent_checks/`
- `reports/strategies/gemini_registry/`
- `reports/strategies/combined/` when the concept is compared against active
  portfolios

Operating rule:

- if a concept is still broad and not yet owned by `125`, `4wind`, `c2`,
  `l3_124`, `l1_234`, or another named logic family, start by referencing it
  here
- once the concept becomes clearly project-specific, add or move the ownership
  note into the matching `projects/<name>/` folder
